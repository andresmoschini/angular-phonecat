AngularJs tutorial

AngularJs is an open-source JavaScript framework useful to create single page applications.

HTML is great for declaring static documents, but it falters when we try to use it for declaring dynamic views in web-applications. AngularJS lets you extend HTML vocabulary for your application. The resulting environment is extraordinarily expressive, readable, and quick to develop.

Proof of concept

Lets start with the end, to show it before boring you. After finish the tutorial, I wanted to test AngularJS in a real environment. I tried to apply it replacing a real part of a real system.

We are working in a HR System, you can play with it here. Well, I have been replacing the old employee search page with a new AngularJS powered employee search page, both look as the same, in fact, the only new features are related to address bar using HTML5 History API AngularJS features and pluralization in result count message. But the code of Angular version is nicer (although not enough nice yet).

In a couple of places I had to modify Angular source code, in others I had to take ugly workarounds:

  • Allow change browser URL without reinitialize controller status
  • Add support to Catch-All routes
  • Property name dependent on View repeater name
  • Using _target to fix link behavior
  • Writing weir code because it is not possible to cancel AJAX requests

I am not as sure about maturity of AngularJS, I think that it is a little difficult to do some common things and documentation is not so clear that it seems to be at the first look. For that reason I prefer to finish my AngularJS self-training here and then, maybe, go deeper in some other Javascript framework like knockout or backbone.

Other considerations

It was really nice and interesting play and discover some useful tools like GitHub Pages and AppHarbor. Both are really powerful tools that we can take advantage in our self-trainings.
